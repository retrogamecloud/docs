## Autenticación y Seguridad

Todos los endpoints de la API están autenticados usando tokens Bearer y están definidos en el archivo de especificación OpenAPI.

```json
"security": [
  {
    "bearerAuth": []
  }
]

```

### Tipos de Autenticación

<AccordionGroup>
  <Accordion title="Bearer Token Authentication">
    Método principal para autenticación de APIs REST
    ```bash
    curl -H "Authorization: Bearer YOUR_TOKEN" https://api.example.com/endpoint
    ```

  </Accordion>
  <Accordion title="API Key Authentication">
    Para integraciones de servicios específicos
    ```bash
    curl -H "X-API-Key: YOUR_API_KEY" https://api.example.com/endpoint
    ```

  </Accordion>
</AccordionGroup>

## Códigos de Respuesta

| Código | Descripción | Uso |
|--------|-------------|-----|
| `200` | OK | Operación exitosa |
| `201` | Created | Recurso creado correctamente |
| `400` | Bad Request | Error en los parámetros de entrada |
| `401` | Unauthorized | Token de autenticación inválido |
| `403` | Forbidden | Permisos insuficientes |
| `404` | Not Found | Recurso no encontrado |
| `429` | Rate Limited | Límite de requests excedido |
| `500` | Internal Error | Error interno del servidor |

## Rate Limiting

Las APIs implementan rate limiting para garantizar la estabilidad del sistema:

- **Requests por minuto**: 1000 requests/min por token

- **Burst limit**: 100 requests en 10 segundos

- **Headers de respuesta**: `X-RateLimit-Remaining`, `X-RateLimit-Reset`

<Warning>
  Los límites pueden variar según el tipo de endpoint y el plan de suscripción.
</Warning>
