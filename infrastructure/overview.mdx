## Infraestructura como Código

La infraestructura de Retro Game Hub está completamente gestionada con **Terraform**, asegurando despliegues reproducibles y versionados.

<Warning>
  **Antes de desplegar**: Completa el [Bootstrap Inicial](/infrastructure/bootstrap) para configurar credenciales AWS, backend remoto y variables.
</Warning>

## Flujo de Despliegue


### Orden de Despliegue Recomendado

<Steps>
  <Step title="Bootstrap (Prerequisito)">
    Configurar backend de Terraform, credenciales AWS, variables
    
    Ver [Guía de Bootstrap](/infrastructure/bootstrap)
  </Step>
  
  <Step title="Terraform Init">
    ```bash
    terraform init
    ```
    Inicializa provider y backend remoto S3
  </Step>
  
  <Step title="Terraform Apply (Una Sola Vez)">
    ```bash
    terraform apply
    ```
    Terraform resuelve dependencias automáticamente:
    1. VPC y networking (5 min)
    2. EKS cluster (10 min)
    3. RDS, S3, CloudFront (5 min)
    4. Monitoring stack (3 min)
    5. ArgoCD (2 min)
  </Step>
  
  <Step title="Configurar kubectl">
    ```bash
    aws eks update-kubeconfig --region eu-west-1 --name retrogame
    ```
  </Step>
  
  <Step title="Configurar ArgoCD Applications">
    ```bash
    kubectl apply -f argocd/apps/
    ```
    ArgoCD ahora gestiona los despliegues
  </Step>
</Steps>

### Despliegue por Fases (Solo para Troubleshooting)

Si necesitas desplegar componentes por separado:

```bash
# Fase 1: Networking
terraform apply -target=module.vpc -target=aws_security_group.eks

# Fase 2: EKS
terraform apply -target=module.eks

# Fase 3: Databases y Storage
terraform apply -target=aws_db_instance.postgres -target=aws_s3_bucket.cdn

# Fase 4: Monitoring y ArgoCD
terraform apply -target=helm_release.kube_prometheus_stack -target=helm_release.argocd

# Fase 5: Todo lo restante
terraform apply
```

<Note>
  El despliegue por fases solo es necesario si encuentras errores. En condiciones normales, `terraform apply` maneja todo.
</Note>

## Estructura de Terraform

```
terraform/eks_test/
├── main.tf                  # Configuración del provider
├── variables.tf             # Variables de entrada
├── outputs.tf               # Valores de salida
├── vpc.tf                   # VPC y redes
├── eks.tf                   # Configuración del cluster EKS
├── rds.tf                   # Base de datos PostgreSQL
├── s3.tf                    # Buckets S3 y CloudFront
├── route53.tf               # DNS y certificados SSL
├── alb.tf                   # Application Load Balancer
├── monitoring.tf            # Prometheus, Grafana, AlertManager
├── oauth2_proxy.tf          # OAuth de GitHub para monitoreo
├── kubernetes.tf            # Deployments de K8s
├── kong.tf                  # Kong API Gateway
└── terraform.tfvars         # Valores de variables (gitignored)
```

## Gestión de Estado

<Info>
El estado de Terraform se almacena remotamente en S3 con bloqueo DynamoDB para permitir colaboración en equipo.
</Info>

```hcl
terraform {
  backend "s3" {
    bucket         = "retrogame-terraform-state"
    key            = "eks-test/terraform.tfstate"
    region         = "eu-west-1"
    dynamodb_table = "terraform-lock"
    encrypt        = true
  }
}
```

## Optimización de Costos

### Costos Actuales (Estimados)
- **Cluster EKS**: $72/mes (plano de control)
- **Instancias EC2**: ~$30-60/mes (nodos t3.medium)
- **RDS**: $15-25/mes (db.t3.micro)
- **NAT Gateway**: $32/mes
- **Transferencia de Datos**: Variable
- **Total**: ~$135-160/mes

<Warning>
Las instancias t3.micro son insuficientes para cargas de trabajo de EKS. Mínimo recomendado: **t3.medium** (4GB RAM, 17 pods/nodo).
</Warning>

### Consideraciones de Free Tier
- Plano de control EKS: **NO** cubierto por Free Tier ($72/mes inevitable)
- Instancias t3.micro: Solo 750 horas/mes = 1 instancia gratis
- RDS: 750 horas/mes db.t3.micro incluidas
- S3: 5GB de almacenamiento estándar gratis
- Transferencia de Datos: 100GB salida por mes gratis

## Flujo de Despliegue

<Steps>
  <Step title="Inicializar Terraform">
    ```bash
    cd terraform/eks_test
    terraform init
    ```
  </Step>
  
  <Step title="Planificar Cambios">
    ```bash
    terraform plan
    ```
  </Step>
  
  <Step title="Aplicar Infraestructura">
    ```bash
    terraform apply
    ```
    Duración: ~20-30 minutos
  </Step>
  
  <Step title="Configurar kubectl">
    ```bash
    aws eks update-kubeconfig --region eu-west-1 --name retrogame
    ```
  </Step>
  
  <Step title="Configurar ArgoCD">
    ```bash
    kubectl apply -f argocd/apps/
    kubectl get applications -n argocd
    ```
    ArgoCD gestiona despliegues desde aquí
  </Step>
  
  <Step title="Verificar Despliegue">
    ```bash
    kubectl get nodes
    kubectl get pods --all-namespaces
    ```
    Todo debe estar en estado Running/Healthy
  </Step>
</Steps>
