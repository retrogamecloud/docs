## Despliegue Paso a Paso

<Steps>
  <Step title="Clonar el Repositorio">
    ```bash
    git clone https://github.com/tu-usuario/retro-game-hub.git
    ```

  </Step>

  <Step title="Configurar Variables">
    Copia y edita el archivo de variables:
    
    ```bash

    ```
    
    Edita las variables principales:
    
    ```hcl

    environment  = "production"
    aws_region   = "eu-west-1"
    
    # Tu dominio (opcional)
    ```

  </Step>

  <Step title="Desplegar Infraestructura">
    ```bash
    cd terraform
    
    # Inicializar Terraform
    terraform init
    
    # Revisar el plan
    terraform plan
    
    # Aplicar cambios
    terraform apply -auto-approve
    ```

    <Warning>
      Este paso toma ~15-20 minutos. El cluster EKS es lo que m√°s tiempo consume.


  <Step title="Acceder a la Aplicaci√≥n">

## Soluci√≥n de Problemas

<AccordionGroup>
  <Accordion icon="warning" title="Error en el despliegue de EKS">
    Si el cluster EKS no se crea correctamente:
    
    ```bash

    aws cloudformation describe-stack-events --stack-name eksctl-retro-game-hub-cluster
    
    # Limpiar y reintentar

    terraform apply -target=module.eks
    ```
  </Accordion>

  <Accordion icon="network-wired" title="Problemas de conectividad">
    Si los pods no pueden comunicarse:
    
    ```bash

    kubectl describe pod [POD_NAME]
    
    # Revisar logs de red
    kubectl logs -l app=aws-load-balancer-controller -n kube-system
    ```
  </Accordion>

  <Accordion icon="database" title="Errores de base de datos">

    ```bash
    aws rds describe-db-instances --db-instance-identifier retro-game-hub-db
    ```
