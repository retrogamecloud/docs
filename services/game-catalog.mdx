## Descripción

El servicio de catálogo gestiona el inventario de juegos retro disponibles en la plataforma, incluyendo metadata, imágenes y archivos .jsdos.

## Información Técnica

| Atributo | Valor |
|----------|-------|
| **Puerto** | 3003 |
| **Base de Datos** | PostgreSQL |
| **Dependencias** | - |
| **Tecnología** | Node.js + Express |

## Funcionalidades Principales

- **Gestión de Catálogo**: CRUD completo de juegos

- **Búsqueda Avanzada**: Por nombre, género, año, desarrollador

- **Metadata Rica**: Descripción, capturas, archivos .jsdos

- **Filtros**: Por categorías y características

- **Integración**: APIs para otros servicios

## Problemas Comunes

<AccordionGroup>
  <Accordion title="Juego no se encuentra en búsqueda">
    **Síntomas:**
    - La API devuelve array vacío
    - Búsqueda por slug no funciona

    **Logs Relevantes:**
    ```bash
    [WARN] Game not found: invalid-slug
    [ERROR] Database query failed: games.title ILIKE '%undefined%'
    ```

    **Soluciones Rápidas:**
    - Verificar que el slug existe en base de datos
    - Comprobar parámetros de búsqueda
    - Revisar conexión a PostgreSQL
    ```bash
    # Verificar juego específico
    curl http://localhost:3003/api/games/doom

    # Listar todos los slugs
    curl http://localhost:3003/api/games?fields=slug
    ```

  </Accordion>

  <Accordion title="Archivos .jsdos corruptos o faltantes">
    **Síntomas:**
    - Error 404 al solicitar archivo de juego
    - Metadata sin URL de descarga

    **Logs Relevantes:**
    ```bash
    [ERROR] File not found: /storage/games/doom.jsdos
    [WARN] Missing jsdos_url for game ID: 1
    ```

    **Soluciones Rápidas:**
    - Verificar integridad de archivos en storage
    - Re-subir archivos .jsdos faltantes
    - Actualizar metadata en base de datos
    ```bash
    # Verificar archivos
    ls -la /storage/games/*.jsdos

    # Verificar metadata
    curl http://localhost:3003/api/games/doom | jq '.jsdos_url'
    ```

  </Accordion>

  <Accordion title="Base de datos desactualizada">
    **Síntomas:**
    - Juegos nuevos no aparecen
    - Metadata inconsistente

    **Logs Relevantes:**
    ```bash
    [ERROR] PostgreSQL connection failed
    [WARN] Table 'games' migration pending
    ```

    **Soluciones Rápidas:**
    - Ejecutar migraciones pendientes
    - Reindexar catálogo
    - Verificar permisos de base de datos
    ```bash
    # Ejecutar migraciones
    npm run migrate

    # Reindexar catálogo
    curl -X POST http://localhost:3003/api/admin/reindex
    ```

  </Accordion>

  <Accordion title="Búsqueda lenta o timeout">
    **Síntomas:**
    - Respuestas lentas en endpoints de búsqueda
    - Timeout en consultas complejas

    **Logs Relevantes:**
    ```bash
    [WARN] Query took 5.2s: SELECT * FROM games WHERE...
    [ERROR] Database timeout after 10s
    ```

    **Soluciones Rápidas:**
    - Verificar índices de base de datos
    - Optimizar consultas SQL
    - Implementar caché de resultados
    ```bash
    # Verificar performance
    curl -w "Time: %{time_total}s\n" http://localhost:3003/api/games/search?q=doom

    # Limpiar caché
    curl -X DELETE http://localhost:3003/api/cache
    ```

  </Accordion>

  <Accordion title="Imágenes y capturas no cargan">
    **Síntomas:**
    - URLs de imágenes devuelven 404
    - Metadata sin screenshots

    **Logs Relevantes:**
    ```bash
    [ERROR] Image not found: /assets/screenshots/doom_1.jpg
    [WARN] Missing thumbnail for game: duke3d
    ```

    **Soluciones Rápidas:**
    - Verificar directorio de assets
    - Re-generar thumbnails faltantes
    - Actualizar URLs en base de datos
    ```bash
    # Verificar assets
    ls -la /assets/screenshots/

    # Regenerar thumbnails
    curl -X POST http://localhost:3003/api/admin/generate-thumbnails
    ```

  </Accordion>
</AccordionGroup>
