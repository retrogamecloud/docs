### Días 2-5: Setup del Entorno Local

<Tabs>
<Tab title="Configuración Backend">

```bash

# Clonar repositorio principal
git clone git@github.com:company/retro-game-cloud.git
cd retro-game-cloud

# Setup de servicios de infraestructura
docker-compose up -d postgres redis

# Verificar que los servicios estén corriendo
docker-compose ps

# Instalar dependencias de microservicios
cd services/auth && npm install
cd ../user-service && npm install
cd ../game-catalog && npm install
cd ../score-service && npm install
cd ../ranking-service && npm install

# Variables de entorno
cp .env.example .env.local

# Configurar variables según documentación

# DATABASE_URL, REDIS_URL, JWT_SECRET, etc.

# Ejecutar migraciones
npm run migrate:dev

# Seed inicial de datos
npm run seed:dev

# Verificar setup
npm run test:health

```

* *Checklist Backend:**

- [ ] PostgreSQL corriendo en puerto 5432

- [ ] Redis corriendo en puerto 6379

- [ ] Todas las dependencias instaladas sin errores

- [ ] Variables de entorno configuradas

- [ ] Migraciones ejecutadas exitosamente

- [ ] Datos de prueba cargados

- [ ] Health checks pasando

</Tab>
<Tab title="Configuración Frontend">

```bash

# Setup del frontend
cd frontend
npm install

# Configurar variables de entorno
cp .env.example .env.local

# Variables principales:

# NEXT_PUBLIC_API_URL=http://localhost:8080

# NEXT_PUBLIC_WS_URL=ws://localhost:8081

# NEXT_PUBLIC_CDN_URL=http://localhost:9000

# Ejecutar en modo desarrollo
npm run dev

# Verificar en http://localhost:3000

```

* *Checklist Frontend:**

- [ ] Dependencias instaladas correctamente

- [ ] Variables de entorno configuradas

- [ ] Aplicación corriendo en localhost:3000

- [ ] Login funcional con usuarios de prueba

- [ ] Navegación básica funcionando

- [ ] Conexión con APIs backend establecida

</Tab>
<Tab title="Configuración DevOps">

```bash

# Setup de kubectl para cluster local
kubectl config use-context docker-desktop

# Verificar acceso al cluster dev
kubectl get pods -n retro-game-dev

# Setup de AWS CLI
aws configure

# Usar credenciales de desarrollo proporcionadas

# Verificar acceso a S3
aws s3 ls s3://retro-game-assets-dev/

# Setup de ArgoCD CLI (opcional)
argocd login argocd.dev.company.com

```

* *Checklist DevOps:**

- [ ] kubectl configurado correctamente

- [ ] Acceso al cluster de desarrollo

- [ ] AWS CLI configurado con credenciales dev

- [ ] Acceso a buckets S3 de desarrollo

- [ ] ArgoCD CLI instalado y configurado

</Tab>
</Tabs>

### Días 6-10: Primera Contribución

<Warning>
No hagas cambios directos en `main`. Siempre trabaja en feature branches y crea Pull Requests.
</Warning>

#### Estructura de Trabajo

```bash

# Crear branch para primera tarea
git checkout -b feature/onboarding-fix-typo

# Hacer cambios, commit y push
git add .
git commit -m "fix: correct typo in user service documentation"
git push origin feature/onboarding-fix-typo

# Crear PR en GitHub

```

* *Tareas de Primera Contribución:**

<Tabs>
<Tab title="Semana 1 (Días 6-10)">

* *Tarea 1: Fix de Documentación (Día 6)**

- [ ] Encontrar y corregir un typo en documentación

- [ ] Crear PR con la corrección

- [ ] Pasar code review

- [ ] Merge exitoso

* *Tarea 2: Test Unitario Simple (Día 7-8)**

- [ ] Agregar test unitario faltante en auth-service

- [ ] Asegurar 100% coverage en función específica

- [ ] Ejecutar tests localmente

- [ ] PR y review

* *Tarea 3: Bug Fix Menor (Día 9-10)**

- [ ] Resolver issue etiquetado como "good-first-issue"

- [ ] Reproducir bug localmente

- [ ] Implementar fix

- [ ] Agregar test regression

- [ ] PR con descripción detallada

</Tab>
<Tab title="Semana 2 (Días 11-15)">

* *Tarea 4: Nueva Feature Pequeña (Día 11-13)**

- [ ] Implementar endpoint GET /api/health en user-service

- [ ] Incluir información de database connection

- [ ] Agregar tests de integración

- [ ] Actualizar documentación API

* *Tarea 5: Refactoring (Día 14-15)**

- [ ] Refactorizar función específica en game-catalog

- [ ] Mejorar legibilidad sin cambiar funcionalidad

- [ ] Mantener todos los tests pasando

- [ ] Agregar JSDoc comments

</Tab>
</Tabs>

#### Code Review Process

* *Como Autor del PR:**

- [ ] Self-review antes de solicitar review

- [ ] Descripción clara del cambio

- [ ] Screenshots si aplica (cambios UI)

- [ ] Tests pasando en CI/CD

- [ ] Resolver comentarios promptamente

* *Como Reviewer (a partir de la semana 3):**

- [ ] Revisar lógica de negocio

- [ ] Verificar tests y coverage

- [ ] Comprobar estándares de código

- [ ] Sugerir mejoras constructivamente

#!/bin/bash

  "dev")
    docker-compose up -d postgres redis
    cd services/auth && npm run dev &
    cd ../user-service && npm run dev &
    cd ../../frontend && npm run dev
    ;;
  "test")
    npm run test:all
    ;;
  "reset")
    docker-compose down
    docker-compose up -d postgres redis
    npm run migrate:reset
    npm run seed:dev
    ;;
esac
