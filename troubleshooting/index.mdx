# 6.10. Resolución de Problemas

Esta guía proporciona soluciones estructuradas para los problemas más comunes en RetroGameCloud, organizadas por categoría para facilitar el diagnóstico y la resolución.

<Note>
Para cada problema, sigue el proceso: **Síntomas** → **Diagnóstico** → **Solución** → **Prevención**
</Note>

## Metodología de Resolución de Problemas

### Proceso Estructurado

<Steps>
<Step title="Identificación">

* *Recopila información inicial:**

- ¿Cuándo comenzó el problema?

- ¿Qué cambios se realizaron recientemente?

- ¿El problema es intermitente o constante?

- ¿Afecta a todos los usuarios o solo a algunos?
</Step>

<Step title="Diagnóstico">

* *Examina los logs y métricas:**

- Revisa logs de aplicación y sistema

- Verifica métricas de rendimiento

- Comprueba el estado de los servicios

- Analiza la conectividad de red
</Step>

<Step title="Solución">

* *Aplica la corrección apropiada:**

- Implementa la solución paso a paso

- Documenta los cambios realizados

- Verifica que el problema se resolvió

- Monitorea para asegurar estabilidad
</Step>

<Step title="Prevención">

* *Implementa medidas preventivas:**

- Actualiza documentación

- Mejora monitoreo y alertas

- Aplica mejores prácticas

- Planifica capacitación del equipo
</Step>
</Steps>

## Herramientas de Diagnóstico

### Comandos Esenciales

<CodeGroup>

```bash Kubernetes

# Estado general del cluster
kubectl get pods -A
kubectl get services -A
kubectl get nodes

# Logs detallados
kubectl logs -f <pod-name> -n <namespace>
kubectl describe pod <pod-name> -n <namespace>

# Eventos del cluster
kubectl get events --sort-by=.metadata.creationTimestamp

```

```bash Aplicación

# Logs de la aplicación
docker logs <container-id> --tail 100 -f

# Estado de servicios
systemctl status <service-name>

# Conectividad de red
ping <hostname>
telnet <hostname> <port>
nslookup <domain>

```

```bash Base de Datos

# Conexión PostgreSQL
psql -h <host> -U <user> -d <database>

# Estado de conexiones
SELECT * FROM pg_stat_activity;

# Verificar locks
SELECT * FROM pg_locks;

```

</CodeGroup>

## Problemas Críticos Comunes

### Aplicación No Disponible

<Accordion title="Diagnóstico Rápido">
<Steps>
<Step title="Verificar Pods">

```bash
kubectl get pods -n retrogame
kubectl describe pod <failing-pod> -n retrogame

```

</Step>

<Step title="Revisar Servicios">

```bash
kubectl get services -n retrogame
kubectl get ingress -n retrogame

```

</Step>

<Step title="Comprobar Logs">

```bash
kubectl logs -f deployment/retrogame-backend -n retrogame
kubectl logs -f deployment/retrogame-frontend -n retrogame

```

</Step>

<Step title="Verificar Conectividad">

```bash

# Test interno del cluster
kubectl run test-pod --image=busybox -it --rm -- /bin/sh
nslookup retrogame-backend.retrogame.svc.cluster.local

# Test externo
curl -I https://your-domain.com

```

</Step>
</Steps>
</Accordion>

### Base de Datos Inaccesible

<Accordion title="Pasos de Resolución">
<Steps>
<Step title="Estado del Pod PostgreSQL">

```bash
kubectl get pods -l app=postgresql -n retrogame
kubectl logs postgresql-0 -n retrogame

```

</Step>

<Step title="Verificar Configuración">

```bash
kubectl get configmap postgres-config -n retrogame -o yaml
kubectl get secret postgres-secret -n retrogame -o yaml

```

</Step>

<Step title="Test de Conectividad">

```bash

# Desde pod de aplicación
kubectl exec -it deployment/retrogame-backend -n retrogame -- \
  psql -h postgresql -U retrogame -d retrogame_db -c "SELECT 1;"

```

</Step>

<Step title="Revisar Recursos">

```bash
kubectl top pods -n retrogame
kubectl describe pvc postgres-pvc -n retrogame

```

</Step>
</Steps>
</Accordion>

### Errores de Autenticación

<Accordion title="Solución Paso a Paso">
<Steps>
<Step title="Verificar JWT">

- Comprobar expiración de tokens

- Validar configuración de secretos

- Revisar algoritmos de firma
</Step>

<Step title="OAuth Issues">

- Verificar configuración de providers

- Comprobar URLs de callback

- Validar client_id y client_secret
</Step>

<Step title="Permisos y Roles">

- Revisar asignación de roles

- Verificar políticas de acceso

- Comprobar middleware de autorización
</Step>
</Steps>
</Accordion>

## Escalado de Problemas

### Matriz de Severidad

| Severidad | Descripción | Tiempo de Respuesta | Escalado |
|-----------|-------------|-------------------|----------|
| **Crítica** | Aplicación completamente inaccesible | 15 minutos | Inmediato al lead técnico |
| **Alta** | Funcionalidad principal afectada | 1 hora | Al equipo senior |
| **Media** | Funcionalidad secundaria afectada | 4 horas | Al equipo de desarrollo |
| **Baja** | Problemas cosméticos o menores | 24 horas | Backlog normal |

- **Almacenamiento**: Uso de disco y I/O

- **Aplicación**: Rate de errores y tiempo de respuesta
